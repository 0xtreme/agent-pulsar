Step 4: Test & Connect

Submit a test task to verify everything works.

With services running, try submitting a task:

curl -X POST http://localhost:8100/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "you",
    "conversation_id": "test",
    "intent": "research.summarize",
    "raw_message": "What is event-driven architecture?",
    "params": {},
    "priority": "normal"
  }'

You'll get a request_id back. Check its status:

curl http://localhost:8100/tasks/<request_id>

You can also try any task — the system handles it dynamically:

curl -X POST http://localhost:8100/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "you",
    "conversation_id": "test",
    "intent": "cooking.find_recipe",
    "raw_message": "Find me a quick pasta recipe for dinner",
    "params": {"cuisine": "italian"},
    "priority": "normal"
  }'

Connect via Telegram

Talk to Agent Pulsar from your phone.

Agent Pulsar uses OpenClaw for messaging. To connect Telegram:

1
Install OpenClaw
Follow the OpenClaw setup guide
2
Register the Agent Pulsar skill
cp -r skills/agent-pulsar/ <openclaw-dir>/skills/
3
Connect Telegram
Open OpenClaw Control UI at http://localhost:18789 and add your Telegram bot token
4
Send a message
Open Telegram, message your bot, and try: "Research the latest AI trends and email me a summary"
⚡

You're all set!

Agent Pulsar is running. You can ask it anything — from research and emails to custom tasks. The system dynamically figures out what to do.

Manage credentials: http://localhost:8102 (Config Portal)
Health check: http://localhost:8100/health
Docs: GitHub
