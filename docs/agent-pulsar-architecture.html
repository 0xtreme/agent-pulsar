1

User sends message via Telegram/WhatsApp

"Run payroll for March for all Easyrun employees"

PA Agent receives via channel adapter. Intent parser classifies as payroll.run. Task decomposer creates atomic tasks: fetch_employee_list, calculate_payroll, submit_payroll, send_payslips. Each gets a dependency order.

2

Tasks published to Kafka backlog

PA publishes 4 tasks to task.backlog.payroll topic. Each message includes: task_id, type, params, priority, dependency chain, and a reference to the scoped credentials it'll need (not the credentials themselves). PA replies to user: "On it — running payroll for March. I'll update you when done."

3

Supervisor detects new tasks

Supervisor consumer group picks up the events. Checks skill registry for healthy workers. Finds Payroll Agent available. Validates dependency order (fetch first, then calculate, etc). Signals Payroll Agent to spin up via container orchestrator.

4

Worker executes, dies

Payroll Agent container spins up. Requests scoped Xero API token from Token Broker (vault issues xero:payroll:write token, TTL 5min). Agent executes the task sequence. Writes result to task.results. Updates task.status → COMPLETED. Container terminates. Token auto-revoked on expiry.

5

Supervisor collects result

Supervisor sees COMPLETED status on task.status. Pulls result from task.results. Enriches with summary (4 employees paid, total $X). Publishes completion event to task.completed topic.

6

PA confirms to user

PA consumer picks up completion event. Formats human-friendly response. Sends via Telegram: "Done — March payroll completed for 4 employees. Total: $12,340. Payslips sent to their emails. Need anything else?"

Credential Isolation

Zero-trust between tiers

API keys are never stored in agent memory or context windows. They live only in the Vault. Workers request scoped tokens via TokenBroker.request(skill, scope, ttl). PA and Supervisor never see raw credentials. Each worker gets the minimum permissions needed for its task.

Short-Lived Tokens

Time-bombed access

Every token issued to a worker has a TTL (default: 5 minutes). If a task takes longer, the worker must re-request. On task completion or failure, tokens are revoked immediately. Even if a container is compromised, the blast radius is a single 5-minute window for a single API.

Container Sandboxing

Blast radius containment

Each worker runs in an isolated Docker container with read-only filesystem, no network access except to specific MCP endpoints, and resource limits (CPU, memory, time). Container is destroyed after task completion. No shared state between worker instances.

Encryption at Rest & In Transit

AES-256 + mTLS

All data in Kafka is encrypted. All inter-service communication uses mTLS. Vault data is encrypted with master key (AES-256-GCM). User conversation data is encrypted before storage. Vector embeddings are stored without PII (stripped at ingestion).

Audit Trail

Immutable logging

Every action is logged to an append-only Kafka topic: who accessed what credential, which API was called, what data was read/written, task lifecycle events. Logs are shipped to cold storage (S3/Blob) for compliance. Users can view their own audit trail.

User Key Onboarding

Secure key submission

Users submit API keys via encrypted channel (HTTPS + client-side encryption before submission). Keys are immediately written to Vault and never stored in application memory or logs. Users can rotate or revoke keys at any time. The PA only stores a key_ref, never the key itself.

Key insight: What none of them do

OpenClaw is the closest to a personal AI assistant, but it runs as a single persistent agent with shared memory — when it handles payroll and email in the same session, context bleeds. Eigent has multi-agent but it's desktop-centric and designed for productivity workflows, not a personal life OS. CrewAI and LangGraph are developer frameworks — great for building specific pipelines, but not designed as an always-on assistant.

Your differentiator: A chat-first personal assistant where heterogeneous tasks are truly isolated, workers are ephemeral (like serverless functions), the message bus provides enterprise-grade reliability (ordering, replay, DLQ), and credentials are vault-isolated with scoped tokens. It's the distributed systems playbook applied to personal AI — and nobody has done exactly this.
